1. Create a Repository on GitHub
Go to GitHub and log in.

Click the "New" button or "New Repository" option.

Fill in:

Repository Name → Example: linux-notes

Description (optional) → e.g., My Linux learning notes and scripts

Visibility → Public or Private

Optionally, check "Add a README file"

Click "Create repository".

2. Clone the Repository Locally
bash
Copy
Edit
git clone https://github.com/your-username/linux-notes.git
This creates a local copy of your repo on your system.

3. Add Your Linux Notes or Scripts
Create or copy your Linux notes/scripts inside the cloned folder:

bash
Copy
Edit
cd linux-notes
nano day1_notes.txt  # Example notes file
Save your file and exit.

4. Initialize Git (if needed)
If you didn’t clone and want to start fresh:

bash
Copy
Edit
git init
5. Stage and Commit Your Files
bash
Copy
Edit
git add .
git commit -m "Added Day 1 Linux notes"
git add . → Stages all files.

git commit -m → Records your changes with a message.

6. Push to GitHub
If cloned:

bash
Copy
Edit
git push origin main
If initialized locally, first connect your repo:

bash
Copy
Edit
git remote add origin https://github.com/your-username/linux-notes.git
git branch -M main
git push -u origin main
7. Verify on GitHub
Go to your repository URL in a browser and confirm your notes/scripts are uploaded.

Tips & Best Practices
Always use clear commit messages.

Create separate files for each learning day → day1.txt, day2.txt, etc.

Keep backups on GitHub for easy access anywhere.
